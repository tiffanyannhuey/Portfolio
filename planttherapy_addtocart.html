Plant Therapy Multi-Add-To-Cart Functionality

This project was an interesting one because I had the opportunity to play with a new feature rolled out from the Miva 9.07 release, native predictive search.

Plant Therapy had an existing customized quick add page that was built in Angular - the code would read the products from a file and perform a fuzzy search of those products. The client wanted to try replacing this code with the new native live search preview as it would perform an improved search of all the products in the database and give the admin better control of searchable fields (code, sku, product description, etc.). Since this client is using a third party, Searchspring, for their global search we were able to leverage the native search functionality for this proof-of-concept.

First, I modified the searchfield_menu_dropdown so that each result contained data for product description, sale pricing, etc. I updated the search field prototype to modify the functionality so that it would not lead to the product page on click; rather, it would append an input into the multi-add-to-cart form. The client has the ability to update the quantity or remove the product from the form.

By default, the native search will pull all active products in the database. Plant Therapy had active master products and variants; however, they only wanted the variants to display in the results (i.e. Lemon Essential Oil 10ml, Lemon Essential Oil 30ml, instead of Lemon Essential Oil). Using Miva's LSK, I ran functions to see if the product being called was associated with a master product, and if not NULL, then display the variant with it's current sale pricing. Additional logic for pricing was run on the variant since Plant Therapy has varying price groups for wholesale, employee, etc.
