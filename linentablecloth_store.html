LinenTablecloth Storefront Functionality

The client wanted to have two separate projects for displaying promos on their storefront.

Storefront Hero Banner Slider

The first project was to convert their hero banner to a slider in order to display multiple promotions. The client wanted the ability to swap out the images/text for each slider without going into the code, and have varying number of banners at any given time. This hero banner slider was built using the Slick Slider JS plugin, Miva Readytheme Images/ Content Sections, and the Miva Limited Source Kit (LSK) API. I assigned a unique identifier to the Readytheme Images for the banner, and then leveraged LSK to load the Images onto the storefront page by their unique identifier. If the Readytheme Image was toggled as active, display the image and load in it's corresponding Readytheme Content section with the same unique identifier.


Storefront Promotion Pre-Scheduling Functionality

The client also wanted the ability to pre-schedule promotions on different areas of their storefront in advance. Specs included:

  • The ability to schedule up to 16 promotions at a time
  • Timezone could be set to store system time (EST)
  • They wanted to cohesively time the promotions with their sales/marketing campaigns
  • They wanted the promotions to dynamically change without going into the code

To build out scheduling functionality, I made use of the Marketing Price Groups capability in the Miva Merchant admin. Price Groups are a marketing tool that allows the admin to set basket/shipping/product discounts, coupons, etc. with the ability to set an active date range but does not natively control any display on the store. I created 16 set Price Groups for each of the promo sections using a separate unique ID, which can be modified by the client at any given time. and Using Miva's LSK API, I ran logic to loop through the designated Price Groups to see if any were active - if they are active Price Groups, another condition was run to see if the system time was currently within the date range and displayed the timed promotional banner if true. If there a currently no active Price Groups, then the logic will show the default promotional settings.
